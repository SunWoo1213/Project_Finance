# Frontend Src 개발 방향성

`frontend/src`는 React 앱의 실제 소스 루트입니다.

## 이 위치의 역할

- `main.jsx`: React 앱 mount 지점
- `App.jsx`: 라우팅과 전역 레이아웃
- `pages`: 라우트 단위 화면
- `components`: 재사용 UI
- `store`: 전역 상태
- `utils`: 순수 함수와 정적 매핑
- `assets`: 번들에 포함되는 정적 자산

## 개발 원칙

`App.jsx`는 라우팅과 공통 레이아웃에 집중합니다. 개별 화면의 데이터 호출과 상세 상태는 `pages`에 둡니다.

페이지에서 반복되는 UI가 보이면 `components`로 분리합니다. 여러 화면에서 같은 포맷팅 로직을 쓰면 `utils`로 이동합니다.

전역 상태는 필요한 만큼만 사용합니다. 인증처럼 여러 화면이 공유하는 상태는 `store`에 두고, 특정 화면의 탭/입력/로딩 상태는 해당 컴포넌트 안에 둡니다.
