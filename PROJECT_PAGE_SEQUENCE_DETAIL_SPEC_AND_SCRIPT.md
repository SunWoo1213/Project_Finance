# AI Invest 화면 순서별 기능 상세 명세 및 발표 대본

문서 목적 : 이 문서는 `PROJECT_FUNCTION_DETAIL_SPEC.md`를 바탕으로 발표용 PPT 제작 순서에 맞춰 화면별 기능 상세 명세와 발표 대본을 정리한 문서이다.

작성 기준 : 실제 프론트엔드 라우트(`/`, `/login`, `/register`, `/category/:type`, `/detail/:ticker`)와 백엔드 API(`GET /api/market/prices`, `GET /api/market/history/{ticker}`, `GET /api/reports/{ticker}`, `POST /api/ai/generate/{ticker}`, 댓글 API)를 기준으로 작성하였다.

발표 흐름 : 사용자는 로그인/회원가입을 통해 인증 상태를 만들고, 메인 대시보드에서 시장을 훑은 뒤, 자산 카테고리와 개별 상세 페이지를 이동하면서 가격 차트, AI 리포트, 댓글 기능을 확인한다.

주의 사항 : 원자재와 암호화폐 구간은 실제 코드 기준으로 원자재 목록은 `/category/commodities`, 금 상세는 `/detail/XAU`, 암호화폐 목록은 `/category/cryptos`, 암호화폐 AI 리포트 예시는 비트코인 `/detail/BTC-USD` 기준으로 설명한다.

## 1. 로그인 화면

화면 경로 : `/login`

화면 목적 : 기존 사용자가 이메일과 비밀번호로 로그인하여 AI 리포트 열람, 댓글 작성, 댓글 좋아요 같은 보호 기능에 접근할 수 있도록 한다.

주요 UI : 이메일 입력 필드, 비밀번호 입력 필드, 로그인 버튼, 회원가입 이동 링크로 구성된다.

입력 검증 : `react-hook-form`과 Zod 스키마를 사용하여 이메일 형식과 비밀번호 입력 여부를 검증한다.

API 연동 : `POST /api/auth/login`으로 `application/x-www-form-urlencoded` 형식의 `username`, `password` 값을 전송한다.

성공 처리 : 백엔드가 반환한 `access_token`과 닉네임을 Zustand 상태와 `localStorage`에 저장하고 홈 화면으로 이동한다.

실패 처리 : 비밀번호 불일치나 존재하지 않는 계정은 입력 필드 오류로 표시하고, 서버 연결 실패는 Toast 메시지로 안내한다.

PPT 설명 문구 : 로그인 화면은 JWT 인증의 시작점이며, 이후 AI 리포트와 커뮤니티 기능을 사용할 수 있는 사용자 상태를 만든다.

발표 대본 : 첫 번째 화면은 로그인 화면입니다. 사용자는 이메일과 비밀번호를 입력해 로그인하고, 서버는 정보를 검증한 뒤 JWT 토큰을 발급합니다. 프론트엔드는 이 토큰을 Zustand와 localStorage에 저장해서 새로고침 후에도 로그인 상태를 유지합니다. 이 인증 상태가 있어야 AI 리포트 열람과 댓글 작성 같은 핵심 기능을 사용할 수 있습니다.

## 2. 회원가입 화면

화면 경로 : `/register`

화면 목적 : 신규 사용자가 이메일, 닉네임, 비밀번호를 등록하여 서비스 계정을 생성할 수 있도록 한다.

주요 UI : 이메일 입력 필드, 닉네임 입력 필드, 비밀번호 입력 필드, 비밀번호 확인 입력 필드, 가입하기 버튼, 로그인 이동 링크로 구성된다.

입력 검증 : 이메일 형식, 닉네임 길이, 비밀번호 최소 길이, 비밀번호 확인 일치 여부를 Zod 스키마로 검증한다.

API 연동 : `POST /api/auth/register`로 회원가입 요청을 전송한다.

백엔드 처리 : 이메일과 닉네임 중복을 검사하고, 비밀번호는 bcrypt 해시로 변환하여 User 테이블에 저장한다.

성공 처리 : 회원가입 완료 Toast를 표시하고 로그인 화면으로 이동한다.

실패 처리 : 이메일 또는 닉네임 중복 오류를 해당 입력 필드에 표시한다.

PPT 설명 문구 : 회원가입 화면은 사용자 계정을 생성하고, 이후 댓글 작성자 표시명으로 사용할 닉네임을 등록하는 화면이다.

발표 대본 : 두 번째는 회원가입 화면입니다. 이메일, 닉네임, 비밀번호를 입력하면 프론트엔드에서 먼저 형식 검증을 수행하고, 백엔드에서는 이메일과 닉네임 중복을 확인합니다. 비밀번호는 평문으로 저장하지 않고 bcrypt 해시로 저장하기 때문에 기본적인 보안 구조를 갖추고 있습니다. 가입이 완료되면 로그인 화면으로 이동해 바로 서비스를 사용할 수 있습니다.

## 3. 메인 대시보드

화면 경로 : `/`

화면 목적 : 사용자가 서비스에 들어왔을 때 주요 시장 지수를 한눈에 확인할 수 있도록 한다.

주요 UI : `주요 지수` 제목, S&P 500, Nasdaq 100, KOSPI, KOSDAQ 카드, 현재가, 변동률, 미니 차트로 구성된다.

데이터 호출 : `GET /api/market/prices`를 호출하고 응답의 `macro` 그룹에서 주요 지수 데이터를 가져온다.

표시 데이터 : 각 지수의 현재가, 전일 대비 변동률, 최근 가격 흐름을 SparklineChart로 표시한다.

상호작용 : 사용자가 지수 카드를 클릭하면 `/detail/{ticker}`로 이동하여 해당 지수 상세 화면을 볼 수 있다.

로딩 상태 : 데이터 호출 중에는 `Loading market data...` 메시지를 표시한다.

PPT 설명 문구 : 메인 대시보드는 글로벌 시장의 현재 흐름을 빠르게 파악하는 첫 화면이다.

발표 대본 : 세 번째 화면은 메인 대시보드입니다. 여기서는 S&P 500, 나스닥 100, 코스피, 코스닥처럼 시장 전체 분위기를 보여주는 주요 지수를 카드 형태로 제공합니다. 각 카드에는 현재가, 변동률, 미니 차트가 들어가고, 카드를 누르면 바로 상세 화면으로 이동합니다. 사용자는 처음 화면에서 시장이 상승 흐름인지 하락 흐름인지 빠르게 판단할 수 있습니다.

## 4. 각 자산별 항목

화면 경로 : 공통 헤더의 `/category/us_top10`, `/category/kr_top10`, `/category/bonds`, `/category/commodities`, `/category/cryptos`

화면 목적 : 사용자가 미국 주식, 한국 주식, 채권, 원자재, 암호화폐 자산군으로 빠르게 이동할 수 있도록 한다.

주요 UI : 상단 Header에 `홈`, `미국 주식`, `한국 주식`, `채권`, `원자재`, `암호화폐` 네비게이션이 표시된다.

인증 상태 UI : 로그인 전에는 로그인/회원가입 버튼을 보여주고, 로그인 후에는 사용자 닉네임과 로그아웃 버튼을 보여준다.

카테고리 구조 : 미국 주식은 `us_top10`, 한국 주식은 `kr_top10`, 채권은 `bonds`, 원자재는 `commodities`, 암호화폐는 `cryptos` 키로 구분된다.

데이터 구조 : 모든 카테고리 목록은 `GET /api/market/prices` 응답에서 해당 그룹만 추출하여 화면에 표시한다.

PPT 설명 문구 : 공통 헤더는 서비스의 주요 자산군을 연결하는 탐색 허브 역할을 한다.

발표 대본 : 네 번째는 자산별 항목 이동 구조입니다. 상단 네비게이션을 통해 미국 주식, 한국 주식, 채권, 원자재, 암호화폐로 이동할 수 있습니다. 모든 카테고리는 같은 화면 컴포넌트를 재사용하지만, URL의 카테고리 키에 따라 다른 데이터 그룹을 보여줍니다. 이 구조 덕분에 화면 구성은 일관되면서도 자산군별 탐색이 가능합니다.

## 5. 미국 주식 세부 항목 Top10 페이지

화면 경로 : `/category/us_top10`

화면 목적 : 미국 대표 시가총액 상위 종목 10개를 목록 형태로 제공한다.

주요 자산 : AAPL, MSFT, NVDA, GOOGL, AMZN, META, BRK-B, LLY, AVGO, TSLA를 지원한다.

주요 UI : 종목명, 티커, 미니 차트, 현재가, 변동률, 시가총액이 각 행에 표시된다.

정렬 기준 : 시가총액 값이 있는 경우 `marketCap` 기준 내림차순으로 정렬한다.

데이터 호출 : `GET /api/market/prices` 응답의 `us_top10` 그룹을 사용한다.

상호작용 : 사용자가 엔비디아 행을 클릭하면 `/detail/NVDA`로 이동한다.

PPT 설명 문구 : 미국 주식 Top10 페이지는 글로벌 대형 기술주와 대표 성장주를 비교 탐색하는 화면이다.

발표 대본 : 다섯 번째 화면은 미국 주식 Top10 목록입니다. 이 화면은 애플, 마이크로소프트, 엔비디아, 알파벳 같은 대표 미국 종목을 보여줍니다. 목록에서는 가격과 변동률뿐 아니라 미니 차트를 함께 제공해 단순 표보다 직관적으로 흐름을 확인할 수 있습니다. 여기서 엔비디아를 선택하면 개별 종목 상세 페이지로 이동합니다.

## 6. 엔비디아 세부 페이지

화면 경로 : `/detail/NVDA`

화면 목적 : 엔비디아의 현재 가격, 등락률, 시가총액, 기간별 가격 차트, AI 리포트, 종목 토론방을 제공한다.

주요 UI : 종목명 `엔비디아`, 티커 `NVDA`, 현재가, 변동률 배지, 시가총액, 기간 선택 버튼, 라인 차트, AI 분석 리포트 영역, 댓글 영역으로 구성된다.

가격 데이터 : `GET /api/market/prices` 전체 응답을 순회하여 `symbol`이 `NVDA`인 데이터를 찾는다.

차트 데이터 : `GET /api/market/history/NVDA?period={period}`를 호출하여 1일, 1개월, 1년, 5년 단위의 가격 흐름을 표시한다.

AI 리포트 영역 : 로그인 여부에 따라 리포트 열람 가능 상태와 잠금 상태가 달라진다.

댓글 기능 : 로그인 사용자는 댓글 작성, 수정, 삭제, 좋아요를 사용할 수 있고, 비로그인 사용자는 댓글 조회만 가능하다.

PPT 설명 문구 : 엔비디아 상세 페이지는 개별 자산 분석 화면의 핵심 구조를 보여주는 대표 예시이다.

발표 대본 : 여섯 번째는 엔비디아 상세 페이지입니다. 상세 페이지에서는 현재가와 변동률을 크게 보여주고, 기간 버튼을 통해 1일, 1개월, 1년, 5년 차트를 전환할 수 있습니다. 아래에는 AI 분석 리포트와 종목 토론방이 이어집니다. 즉, 이 화면 하나에서 시세 확인, AI 분석 확인, 사용자 의견 교환이 모두 이루어집니다.

## 7. 비로그인 시 AI 리포트를 볼 수 없는 페이지

화면 경로 : `/detail/NVDA` 비로그인 상태

화면 목적 : AI 리포트를 보호 기능으로 제한하고, 사용자가 로그인하도록 유도한다.

접근 제어 : `authToken`이 없으면 리포트 데이터 요청을 하지 않고 `report` 상태를 비운다.

주요 UI : AI 분석 리포트 영역이 흐림 처리되고, 중앙에 `로그인하고 AI 리포트 보기` 버튼이 표시된다.

상호작용 : 버튼을 누르면 `/login`으로 이동한다.

허용 기능 : 비로그인 상태에서도 가격 정보, 차트, 댓글 목록 조회는 가능하다.

제한 기능 : AI 리포트 열람, 댓글 작성, 댓글 좋아요는 제한된다.

PPT 설명 문구 : 비로그인 상태에서는 핵심 분석 콘텐츠를 잠금 처리하여 인증 기반 기능임을 명확히 보여준다.

발표 대본 : 일곱 번째 화면은 비로그인 상태의 AI 리포트 영역입니다. 가격과 차트는 누구나 볼 수 있지만, AI 분석 리포트는 로그인한 사용자만 볼 수 있도록 제한했습니다. 화면에서는 리포트 영역을 흐림 처리하고 로그인 버튼을 중앙에 배치해서 사용자가 자연스럽게 로그인 흐름으로 이동하도록 했습니다. 이 부분이 JWT 인증과 프론트엔드 접근 제어가 만나는 지점입니다.

## 8. 로그인 시 볼 수 있는 AI 리포트 페이지

화면 경로 : `/detail/NVDA` 로그인 상태

화면 목적 : 로그인 사용자에게 엔비디아 AI 투자 분석 리포트를 제공한다.

리포트 조회 : `GET /api/reports/NVDA`를 JWT Bearer 토큰과 함께 호출한다.

리포트 자동 생성 : 조회 결과가 404이면 `POST /api/ai/generate/NVDA`를 호출하여 리포트를 생성한 뒤 다시 조회한다.

리포트 렌더링 : `AIReport.final_content`를 `react-markdown`으로 렌더링한다.

AI 생성 구조 : 백엔드는 가격 데이터, 뉴스 데이터, 재무 데이터, 매크로 데이터를 LangGraph 파이프라인으로 통합해 한국어 리포트를 생성한다.

저장 구조 : 생성된 리포트는 `AIReport` 테이블에 저장되어 이후 조회 시 재사용할 수 있다.

PPT 설명 문구 : 로그인 후 AI 리포트 페이지는 프로젝트의 차별화 기능인 AI 기반 투자 분석 결과를 보여준다.

발표 대본 : 여덟 번째는 로그인 후 볼 수 있는 AI 리포트입니다. 사용자가 로그인하면 프론트엔드는 JWT 토큰을 포함해 리포트 조회 API를 호출합니다. 리포트가 이미 있으면 바로 보여주고, 없으면 생성 API를 호출해 새 리포트를 만든 뒤 다시 조회합니다. 리포트 본문은 마크다운 형태로 저장되기 때문에 제목, 요약, 분석 내용을 읽기 좋은 형태로 렌더링할 수 있습니다.

## 18. 암호화폐 AI 리포트

화면 경로 : `/detail/BTC-USD` 로그인 상태의 AI 리포트 영역

화면 목적 : 비트코인 같은 암호화폐 자산에 대한 AI 투자 분석 리포트를 제공한다.

리포트 조회 : `GET /api/reports/BTC-USD`로 최신 리포트를 조회한다.

리포트 생성 : 기존 리포트가 없으면 `POST /api/ai/generate/BTC-USD`로 리포트를 생성한다.

분석 맥락 : 가격 변동률, 거래량, 시장 심리, 규제 뉴스, 금리와 유동성 환경, 위험자산 선호도 등이 분석에 활용된다.

AI 처리 차이 : 암호화폐는 기업 재무제표가 없으므로 재무 분석은 생략하고 뉴스, 매크로, 온체인 또는 시장 보조 데이터 중심으로 분석한다.

결과 표시 : 최종 리포트는 한국어 Markdown으로 렌더링되며, 로그인 사용자만 열람할 수 있다.

PPT 설명 문구 : 암호화폐 AI 리포트는 변동성이 큰 디지털 자산을 뉴스와 매크로 맥락까지 포함해 해석하는 기능이다.

발표 대본 : 마지막 화면은 암호화폐 AI 리포트입니다. 비트코인은 기업 재무제표가 없기 때문에 주식과 같은 방식으로 분석하지 않고, 가격 변동률, 거래량, 규제 뉴스, 유동성 환경 같은 요소를 중심으로 분석합니다. 리포트 조회 방식은 다른 자산과 동일하게 JWT 인증을 요구하고, 리포트가 없으면 생성 API를 통해 새로 만듭니다. 이렇게 AI Invest는 자산군별 특성을 반영하면서도 일관된 사용자 경험을 제공합니다.

## 발표 마무리 요약

서비스 요약 : AI Invest는 주요 지수, 미국 주식, 한국 주식, 채권, 원자재, 암호화폐를 한 서비스에서 탐색하고, 개별 자산의 가격 차트와 AI 리포트, 댓글 토론을 제공하는 금융 정보 웹 애플리케이션이다.

기술 요약 : 프론트엔드는 React Vite 기반 SPA로 구성되고, 백엔드는 FastAPI와 PostgreSQL을 사용하며, 외부 금융 API와 LangGraph 기반 AI 파이프라인을 통해 데이터를 수집하고 분석한다.

차별점 요약 : 단순 시세 조회를 넘어 자산별 특성에 맞는 AI 분석 리포트와 사용자 토론 기능을 함께 제공한다는 점이 핵심 차별점이다.

최종 발표 멘트 : 지금까지 화면 순서대로 AI Invest의 주요 기능을 설명드렸습니다. 이 프로젝트는 다양한 금융 자산을 한곳에서 확인할 수 있게 하고, AI 리포트를 통해 사용자가 가격 뒤에 있는 뉴스와 거시경제 맥락까지 이해할 수 있도록 돕는 것을 목표로 합니다.
