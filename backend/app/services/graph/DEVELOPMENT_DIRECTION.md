# LangGraph 리포트 파이프라인 개발 방향성

이 폴더는 AI 리포트 생성 그래프의 내부 구현을 담당합니다.

## 현재 책임

- `state.py`: 그래프 상태 정의
- `nodes.py`: Bull/Bear/Synthesizer 등 노드 로직
- `tools.py`: 리포트 생성을 위한 보조 도구
- `llm.py`: LLM 클라이언트 구성
- `graph.py`: 노드 연결과 실행 그래프 구성

## 개발 원칙

프롬프트, LLM 호출, 그래프 상태 변경은 이 폴더 안에서 관리합니다. API 라우터나 프론트엔드는 그래프 내부 구조를 몰라야 합니다.

리포트 생성 결과는 DB에 저장 가능한 결정적 구조로 반환해야 합니다.

LLM 호출은 비용과 지연이 큰 작업입니다. 사용자 화면 요청마다 무조건 호출하는 방식보다, 스케줄러 또는 명시적 생성 API를 통해 생성하고 조회 API는 DB에서 읽는 방향을 우선합니다.

새 에이전트를 추가할 때는 역할을 명확히 구분합니다. 예를 들어 리스크 검토, 뉴스 요약, 밸류에이션 검토처럼 입력과 출력이 분명해야 합니다.
